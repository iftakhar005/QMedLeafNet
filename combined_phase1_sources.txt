A More Novel Hybrid Quantum-Classical Project for Bangladeshi Medicinal Plant Health Detection
Executive summary
The uploaded proposal already points in the right direction by framing a parameter-efficient hybrid quantum-classical classifier for medicinal leaf analysis. The strongest way to increase novelty, however, isnotto treat the paper as “CNN backbone plus one quantum head,” but to turn it into arigorous benchmark-and-methodology papercentered on three ideas:leakage-safe evaluation,species-conditioned health prediction, andparameter-matched quantum-versus-classical comparisons under realistic deployment constraints. That repositioning is important because the new Bangladesh medicinal-leaf dataset is unusually valuable but also unusually easy to evaluate incorrectly if the provided augmented data are allowed to leak across splits; the dataset page says the set contains1,981 unique original images,20,019 augmented images, seven medicinal plant species, multiple condition classes, and derived augmented folders generated from originals. fileciteturn0file0[1]
The most defensible central hypothesis is this:when compact, biologically meaningful leaf representations are compressed into a very small bottleneck and the health classifier is conditioned on plant species, low-depth variational quantum circuits with data re-uploading can improve macro-F1 and calibration on minority health classes relative to flat dense heads at similar trainable parameter counts, even if top-1 accuracy gains are modest. This is a better hypothesis than “quantum beats classical,” because recent benchmarking work found that, on small classical datasets, out-of-the-box classical models often outperform quantum classifiers and that removing entanglement can sometimes help rather than hurt. That benchmark result makesfairness, parameter matching, and strong classical baselinespart of the novelty, not just methodological hygiene.[2]
The flagship revised project should therefore be framed as aspecies-conditioned, leakage-safe, noise-aware hybrid benchmark for medicinal plant health detection, with lightweight CNN backbones, multiple 4/6/8-qubit VQC heads, matched classical controls, calibration and energy reporting, and one small real-device or calibration-driven noisy-simulation study. This also differentiates your work from recent agricultural HQNN papers that report very high accuracies on rice leaves, from plant-biotechnology comparisons where classical MLPs remained highly competitive, and from the recent medicinal-plant paperMediFlora-Net, which focuses on plant identification using multimodal/quantum-assisted feature extraction rather thanBangladesh-specific medicinal plant health-condition classification under leakage-safe grouped splits.[3]
The revised novelty claim
The core research hypothesis
A strong hypothesis for the paper is:
A species-conditioned hybrid classifier that projects pretrained CNN leaf embeddings into a very small bottleneck and applies low-depth, locality-aware VQCs with repeated data encoding will outperform parameter-matched classical heads on macro-F1, minority-class recall, and calibration under grouped parent-image splits, while keeping trainable parameters and deployment bandwidth low.
This hypothesis is grounded in four facts from the literature. First, the Bangladesh dataset mixesspecies identityandleaf conditioninformation, so auxiliary species prediction is a meaningful inductive bias rather than an artificial add-on. Second, repeated data encoding increases the expressive power of variational quantum models. Third, local observables and shallow/local circuit structure improve trainability relative to globally averaged costs. Fourth, the fair comparison question is now central in QML, because weak baselines can make hybrid models look better than they really are.[4]

Model family and matched baselines
Backbone menu
Use a lightweight backbone family, not one heavy model only. The best primary backbone for the paper isMobileNetV3-Small, because TorchVision reports2,542,856 parametersand0.06 GFLOPS, making it clearly edge-oriented. KeepMobileNetV2as a second lightweight backbone at3,504,872 parametersand0.30 GFLOPS, andEfficientNet-B0as an accuracy-oriented lightweight backbone at5,288,548 parametersand0.39 GFLOPS. A single heavier reference such asResNet18at11,689,512 parametersand1.81 GFLOPSis useful as a sanity-check upper baseline, not as the main deployment model.[7]
For feature dimensions, use the backbone outputsafter global average pooling and before the original classifier. For ResNet18 this is 512 dimensions because the source code definesself.fc = nn.Linear(512 * block.expansion, num_classes)andBasicBlock.expansion = 1. For MobileNetV2 it is 1280 dimensions (self.last_channel). For MobileNetV3-Small the post-feature tensor is 576 channels before its classifier MLP, and the classifier’s penultimate hidden size is 1024. EfficientNet-B0 uses a 1280-dimensional last channel.[8]

TheRing-SEimplementation is easy to reproduce because PennyLane documentsStronglyEntanglingLayersas using a weight tensor of shape(L, M, 3)overMwires, with default two-qubit entanglers such asCNOT, and a range hyperparameter that naturally supports ring-like patterns. Use two layers so the VQC parameter counts remain small: 24, 36, and 48 parameters for 4, 6, and 8 qubits respectively.[10]
TheReUpload-Linearhead is the most important one scientifically. Pérez-Salinas et al. showed that repeated data re-uploading is the key ingredient allowing small quantum classifiers to represent more complex decision functions, and Schuld, Sweke, and Meyer showed that repeated encoding enriches the accessible Fourier spectrum of quantum models. This makes a two-cycle re-uploading head a much better “novelty carrier” than a one-shot variational head.[11]
TheLocal-TTNhead is not meant to claim superiority because of “quantumness.” Its purpose is to test the trainability hypothesis. Cerezo et al. showed that local cost functions can avoid the exponential gradient suppression associated with global costs, at least in the relevant shallow/local regime. A local, shallow, tree-structured head therefore belongs in the benchmark even if it is not the final best model.[12]
Measurement and PyTorch integration
For all VQCs, measure one Pauli-Z expectation per qubit and feed the resulting-vector into a linear readout for the condition logits. Use PennyLaneTorchLayerso the QNode integrates into a normal PyTorch training stack. PennyLane’s documentation explicitly supports conversion of a QNode into a TorchModule, which makes a clean, reproducible hybrid implementation straightforward.[13]

Data handling and experimental protocol
The split policy that avoids leakage
This section is where your paper can become much stronger than typical agricultural image papers. The dataset page states that theoriginal dataset contains entirely unique leaf samples, withone image per leaf, and that theaugmented datasetis derived from those images. That means the correct grouping unit isthe parent original image ID, not the final augmented filename. The safest protocol is therefore:
Useonly original or background-removed original imageswhen defining groups and splits.
Create a uniqueparent_leaf_idfor every original image.
Assign every augmentation of that parent image to the same group.
Generate your own augmentationsafter splitting, or if you use the dataset’s augmented folder at all, only include augmentations whose parent belongs to the training fold.[1]
Because the dataset was collected using detached leaves on relatively uniform backgrounds, it is also important to say directly that this isnot yet an in-field benchmark. Any mobile or field generalization claim must therefore be framed as a future or supplementary experiment, not as a property already demonstrated by the dataset.[18]
Recommended split sizes
The best default protocol is:
Held-out grouped test set:15% of original images, about297 leaves
Development pool:remaining 85%, about1,684 leaves
Grouped stratified 5-fold cross-validation on the development pool:each fold roughly1,347 train / 337 validationoriginals
This yields a stable final test set while still allowing robust model selection. If compute becomes a bottleneck, a fallback design is a single grouped70/15/15split over originals, which is approximately1,387 / 297 / 297originals. The test set should contain only parent-original leaves, never pre-generated augmentations.[19]
Cross-validation, optimization, and augmentation
Usegrouped stratificationwith the finest feasible label granularity. If the metadata supports enough samples per combination, stratify byspecies × condition. If some combinations are too rare, stratify by condition and monitor species balance manually. Train each model withthree random seeds per foldso that every reported number has a fold variance and a seed variance.
A practical training schedule is:
Stage one:freeze the full backbone; train bottleneck, auxiliary species head, conditioning layer, and classifier head
Stage two:unfreeze the last backbone stage only; continue with a lower learning rate
Optimizer:AdamW for neural models
Quantum initialization:use a reduced-domain initialization for trainable circuit angles rather than full, because reduced-domain initialization has been shown to improve trainability of parameterized quantum circuits.[20]
For augmentation, use onlytrain-only online augmentation. A restrained policy is enough:
random rotation up to about 20 degrees
horizontal and vertical flipping
mild brightness/contrast jitter
small Gaussian noise
occasional sharpening or blur
Donotuse aggressive cutout or heavy color perturbation that can erase disease symptoms. The dataset authors already document rotation, flips, brightness/contrast changes, Gaussian noise, and sharpening as part of their augmentation pipeline, which supports using the same family of transforms online while keeping them training-only.[1]
Preprocessing, imbalance handling, and metrics
The dataset provides abackground-removedversion at1440 × 1080, which is the right source for core experiments because it reduces spurious background cues. Resize to224 × 224for the ImageNet-pretrained backbones using the normalization conventions from TorchVision. Keep aspect ratio by padding rather than center-cropping too aggressively, because leaf shape is informative.[21]
For imbalance, use aclass-balanced samplerover condition classes and weighted cross-entropy. If species × condition combinations are the final flat classes, use a capped inverse-frequency weighting so rare combinations are not ignored while extremely rare cases do not destabilize training.

Treat fold-wise p-values as secondary, because cross-validation folds are not truly independent.

Edge, Green-AI, robustness, and explainability
What you can honestly claim about deployment
The paper can make a credibleedge-assistedclaim, but not a strong “fully on-device quantum inference” claim. ONNX Runtime’s mobile documentation emphasizes that models must fit the device’s disk and memory constraints, while Google’s LiteRT benchmark tools provide realistic measurements for initialization time, warmup-state inference time, steady-state inference time, and memory usage on Android and iOS. These tools should be used to benchmark theclassical portionof the pipeline.[23]
The practical deployment architecture should be framed assplit inference:
the phone or edge device runs the lightweight CNN backbone and bottleneck
it transmits only a tiny-dimensional bottleneck vector plus metadata
the remote server runs either the classical shallow head, the quantum simulator, or the real QPU-backed head
This framing is far more realistic than pretending that current mobile devices execute quantum circuits natively. It also creates a useful communications-efficiency story: transmitting4, 6, or 8 floatsis dramatically cheaper than transmitting full images, while preserving privacy better than raw-image upload.[23]
A realistic Green-AI protocol
For training energy, useCodeCarbonplus aGreen Algorithms-style reporting section. CodeCarbon’s methodology explicitly computes energy aspower × time, reads low-level counters such as RAPL when available, and falls back to TDP-based estimates when hardware counters are unavailable. That fallback can matter: CodeCarbon also assumes about50% of TDPas average CPU power in one fallback path, and recent validation work reports that such estimation tools can still err by as much as roughly40%in some settings. That means your energy section should emphasizerelative comparisonsandmeasurement caveats, not false precision.[24]
A good reporting bundle is:
kWh per training run
kgCO₂e per training run
Joules per test image
energy-normalized macro-F1
latency-normalized macro-F1
The strongest Green-AI claim you can likely support is:frozen-backbone, small-bottleneck hybrid heads reduce trainable-state size and data-transfer size, and they can be benchmarked under a transparent energy protocol. Avoid stronger claims such as “quantum is greener” unless you have direct measurements that include queueing, transpilation, and remote-system overhead.[25]
Noise-aware simulation and a small real-device study
For noisy simulation, use Qiskit Aer’s noise module and backend-derived calibration data. IBM’s documentation states that backend properties update after calibrations and can be used to construct noise models for classical simulation, and the Qiskit Aer documentation describes backend-derivedNoiseModelconstruction and custom readout/gate noise insertion. This gives you a strong, current, reproducible way to evaluate whether the best hybrid head survives realistic noise.[26]
A good plan is:
train all models in ideal simulation
evaluate the top two VQC heads under finite-shot settings such as 128, 512, and 1024 shots
evaluate them again with a backend-derived noise model from a current public backend with at least 8 usable qubits
run atiny real-device trialonly on the best4-qubithead and a small subset of the test set, reporting degradation rather than trying to claim hardware superiority
Because exact statevector simulation usescomplex amplitudes, Qiskit notes that even 8-qubit statevectors are trivial in memory on normal hardware; that makes 4/6/8-qubit ablations computationally affordable. The bottleneck in real-device experiments will therefore bedevice noise, transpilation, and queue latency, not the nominal qubit count itself.[27]
Robustness and explainability extensions
For robustness, includeFGSMandPGDattacks. FGSM remains the canonical fast attack, and PGD remains the standard stronger first-order adversary baseline. Even if your contribution is not security-focused, this experiment is valuable because plant symptoms can be small, texture-heavy, and vulnerable to tiny perturbations.[28]
For explainability, use two complementary families:
Grad-CAMon the CNN backbone, because it localizes image regions important for a class decision
Integrated Gradientsfor input attribution, because it satisfies strong attribution axioms and is simple to implement
Quantum interpretabilitythrough input-angle saliency, parameter saliency, and Fourier-spectrum analysis of the learned encoding, motivated by the result that common variational quantum models can be analyzed as partial Fourier series whose accessible frequencies depend on data encoding.[29]
The best novelty-boosting transfer-learning extension is to compare standard ImageNet-pretrained backbones to a frozenDINOv2feature extractor. DINOv2 explicitly argues that self-supervised visual features can act as strong general-purpose features and that simple classifiers on top of them can already perform well. If the DINOv2-plus-classical baseline is very strong, that is useful scientific context for interpreting any hybrid gain.[30]
Expected outcomes, failure modes, and publication strategy
What results are realistic
The most realistic expected outcome isnota dramatic top-1 accuracy jump. A more believable and more publishable result is:
MobileNetV3-Small and MobileNetV2 remain the strongest deployment backbones
ReUpload-Linearbecomes the best VQC family
the4- or 6-qubitsetting outperforms the 8-qubit setting in stability-per-cost
species conditioning improvesmacro-F1more than raw accuracy
the hybrid model’s best gains appear inminority-condition recallandcalibration
strong classical controls, especially theDense-MatchedandFourier-featureheads, remain very competitive
If that is what happens, the paper is still strong. In the current QML climate, a rigorous result showing “hybrid helps here, but only under matched controls and mostly in minority-class behavior or calibration” is more credible than a sweeping promise of quantum advantage.[31]
Failure modes to anticipate early
The main failure modes are straightforward.
The first isno hybrid advantage at allonce the split is leakage-safe and baselines are strong. If that happens, the project can still publish as a benchmarking and negative-results paper, especially if the protocol is tighter than prior work.[5]
The second isdataset realism. The dataset uses detached leaves on relatively uniform backgrounds; if a reviewer asks about field deployment, you must explicitly scope the main result to controlled-image health recognition and, if possible, include a small out-of-distribution background stress test.[18]
The third isquantum trainability collapseat 8 qubits. If the 8-qubit models become unstable, do not force them into the headline figure. Report them honestly as evidence that trainability and noise, not nominal qubit count, are the limiting factors. This is aligned with the barren-plateau literature and strengthens the paper rather than weakening it.[12]
The fourth isoverclaiming deployment. If the VQC runs on a remote simulator or QPU, call the systemedge-assistedorsplit inference, notfully on-device quantum inference. The latter formulation is not realistic today.[32]

Open questions and limitations
A few aspects remain genuinely uncertain and should be stated explicitly.
The dataset page lists condition labels such as healthy, diseased, chlorotic, dried, distorted, insect-affected, mild disease, and growth stages such as young and mature, but theexact final condition ontology and per-class counts must be verified from the metadata CSV before you lock the final label space. Likewise, because the dataset consists of detached leaves on relatively uniform backgrounds, field generalization remains an open question. Finally, any energy claim about remote QPU inference will remain approximate unless the full stack—including queueing and provider-side overhead—is measured directly.[35]
The bottom line is that thebest revised project is a rigorous, species-conditioned, leakage-safe hybrid benchmark with balanced classical controls and deployment-aware evaluation. If the quantum heads win only narrowly, that can still be a strong paper. If they do not win, the work can still make a meaningful contribution by establishing the first careful evaluation protocol for this Bangladesh medicinal-plant health dataset and clarifying exactly where hybrid quantum-classical methods do and do not help.[36]

Step-by-Step Execution Plan for the Revised Hybrid Quantum-Classical Medicinal Plant Health Project
Executive summary
This execution plan turns the revised project into asequenced research programwith hard gates at each stage, so that you do not spend weeks on quantum modeling before the dataset, split policy, and classical baselines are locked down correctly. The central idea from the revised project document is to treat the work as aleakage-safe, species-conditioned, parameter-matched benchmarkrather than a simple “CNN plus quantum head” paper. That means the project should advance in the following order: dataset audit and parent-image grouping; leakage-safe grouped splits; backbone and classical baseline lock-in; bottleneck and species-conditioning implementation; VQC head sweeps at 4, 6, and 8 qubits; calibration and statistical testing; then only after that, noise-aware simulation, edge/Green-AI evaluation, robustness, and explainability. The dataset-specific constraints that drive this order are important: the Mendeley dataset contains1,981 original images,20,019 augmented images,seven species, and derived augmented folders, so evaluation can be badly inflated if augmented siblings leak across train and test. fileciteturn1file0
The most efficient way to complete the project is to operate withtwo tiers of experimentation. First, run a fast screening stage with the primary backboneMobileNetV3-Small, grouped splits, and classical/quantum heads on a tiny bottleneck space. Second, expand only the best-performing configurations to additional backbones, finite-shot and noisy simulations, and the deployment-oriented measurements. This is consistent with the revised document’s emphasis on fair comparisons, parameter matching, careful calibration, and realistic deployment caveats rather than overclaiming quantum advantage. fileciteturn1file0
A practical end state for a strong conference paper is: one held-out grouped test set; grouped 5-fold development cross-validation; classical baselines including logistic regression, SVMs, dense matched heads, and Fourier-feature controls; VQC heads including Ring-SE, ReUpload-Linear, and Local-TTN at 4/6/8 qubits; temperature-scaled calibration; paired statistical tests; latency and energy reporting; then one small noisy-simulation and real-device appendix. A journal version should extend that with robustness, explainability, and preferably a small out-of-distribution or field-style evaluation. fileciteturn1file0
Roadmap and milestone gates
The project is best managed as a set ofmilestone-gated phases. The gate at the end of each phase should be treated as non-negotiable. If a gate fails, the right action is to fix the upstream issue rather than advancing and accumulating technical debt.

**If Phase 1 is done incorrectly:**

**If Phase 1 is done correctly:**

### Objective 5: Create Leakage-Safe Grouped Splits
**Timeline:** Days 9-10
**Goal:** Split the data in a way that prevents augmentation leakage

**The Split Strategy:**

1. **Test Set:** 15% of original parents (~297 parents)

   - No augmentations in test set (use originals only)

   - Used for model selection and hyperparameter tuning
   - Split into 5-fold cross-validation

   - Generate augmentations **AFTER** splitting
   - Augmentations only in training loop
   - Validation and test: resize/pad/normalize only (no augmentation)

```python
#!/usr/bin/env python3
"""
Comprehensive dataset audit for the medicinal plant health detection project.
Verifies file counts, image integrity, and creates baseline statistics.
"""

import os
import json
import hashlib
from pathlib import Path
from collections import defaultdict
import numpy as np
from PIL import Image
from tqdm import tqdm
import pandas as pd

def compute_file_hash(filepath, algorithm='sha256'):
    """Compute hash of a file."""
    hash_obj = hashlib.new(algorithm)
    with open(filepath, 'rb') as f:
        for chunk in iter(lambda: f.read(4096), b''):
            hash_obj.update(chunk)
    return hash_obj.hexdigest()

def audit_directory_structure(raw_data_path):
    """Audit the directory structure."""
    audit_report = {}
    
    for folder in os.listdir(raw_data_path):
        folder_path = os.path.join(raw_data_path, folder)
        if os.path.isdir(folder_path):
            files = [f for f in os.listdir(folder_path) if os.path.isfile(os.path.join(folder_path, f))]
            audit_report[folder] = {
                'file_count': len(files),
                'sample_files': files[:5]
            }
    
    return audit_report

def check_image_integrity(raw_data_path, output_report_path):
    """Check all images for corruption and collect statistics."""
    
    image_stats = {
        'total_files': 0,
        'valid_images': 0,
        'corrupted_images': 0,
        'by_folder': defaultdict(lambda: {'valid': 0, 'corrupted': 0}),
        'size_histogram': defaultdict(int),
        'format_distribution': defaultdict(int),
        'corrupted_files': []
    }
    
    # Walk through all image files
    for root, dirs, files in os.walk(raw_data_path):
        for filename in tqdm(files, desc=f"Checking {os.path.basename(root)}"):
            if not filename.lower().endswith(('.jpg', '.jpeg', '.png', '.gif', '.bmp')):
                continue
            
            filepath = os.path.join(root, filename)
            image_stats['total_files'] += 1
            folder_name = os.path.basename(root)
            
            try:
                # Try to open and validate the image
                img = Image.open(filepath)
                img.verify()
                
                # Reopen after verify (verify closes the file)
                img = Image.open(filepath)
                width, height = img.size
                size_key = f"{width}x{height}"
                
                image_stats['valid_images'] += 1
                image_stats['by_folder'][folder_name]['valid'] += 1
                image_stats['size_histogram'][size_key] += 1
                image_stats['format_distribution'][img.format] += 1
                
            except Exception as e:
                image_stats['corrupted_images'] += 1
                image_stats['by_folder'][folder_name]['corrupted'] += 1
                image_stats['corrupted_files'].append({
                    'file': filepath,
                    'error': str(e)
                })
    
    return image_stats

def create_audit_report(raw_data_path, output_dir):
    """Create comprehensive audit report."""
    
    print("\n" + "="*80)
    print("DATASET AUDIT STARTING")
    print("="*80 + "\n")
    
    # Step 1: Directory structure
    print("Step 1: Auditing directory structure...")
    dir_audit = audit_directory_structure(raw_data_path)
    print(json.dumps(dir_audit, indent=2))
    
    # Step 2: Image integrity
    print("\nStep 2: Checking image integrity (this may take a few minutes)...")
    image_stats = check_image_integrity(raw_data_path, output_dir)
    
    # Step 3: Create report
    report = {
        'audit_date': pd.Timestamp.now().isoformat(),
        'raw_data_path': str(raw_data_path),
        'directory_structure': dir_audit,
        'image_statistics': {
            'total_files_checked': image_stats['total_files'],
            'valid_images': image_stats['valid_images'],
            'corrupted_images': image_stats['corrupted_images'],
            'corruption_rate': image_stats['corrupted_images'] / max(1, image_stats['total_files']),
            'size_distribution': dict(sorted(image_stats['size_histogram'].items())),
            'format_distribution': dict(image_stats['format_distribution']),
            'by_folder': dict(image_stats['by_folder']),
            'corrupted_files': image_stats['corrupted_files'][:10]  # First 10 for reference
        }
    }
    
    # Save report
    output_path = os.path.join(output_dir, 'data_audit.json')
    with open(output_path, 'w') as f:
        json.dump(report, f, indent=2)
    
    print("\n" + "="*80)
    print("AUDIT REPORT SUMMARY")
    print("="*80)
    print(f"Total files checked: {report['image_statistics']['total_files_checked']}")
    print(f"Valid images: {report['image_statistics']['valid_images']}")
    print(f"Corrupted images: {report['image_statistics']['corrupted_images']}")
    print(f"Corruption rate: {report['image_statistics']['corruption_rate']:.2%}")
    print(f"\nImage size distribution:")
    for size, count in sorted(report['image_statistics']['size_distribution'].items()):
        print(f"  {size}: {count} images")
    print(f"\nFormats found: {list(report['image_statistics']['format_distribution'].keys())}")
    print(f"\nFull report saved to: {output_path}")
    print("="*80 + "\n")
    
    return report

if __name__ == "__main__":
    raw_data_path = "data/raw"
    output_dir = "reports"
    
    os.makedirs(output_dir, exist_ok=True)
    report = create_audit_report(raw_data_path, output_dir)
```

```python
"""
Extract metadata from filenames and any provided metadata CSV.
Build the definitive label ontology for the project.
"""

import os
import re
import pandas as pd
from pathlib import Path
from collections import defaultdict
from tqdm import tqdm

class MetadataExtractor:
    """Extract metadata from the medicinal plant dataset."""
    
    def __init__(self, raw_data_path):
        self.raw_data_path = raw_data_path
        self.filename_patterns = self._build_patterns()
        self.metadata = []
        
    def _build_patterns(self):
        """
        Build regex patterns for filename parsing.
        Adjust these based on actual filename format from the dataset.
        
        Expected formats might be:
        - [species]_[condition]_[id]_[augmentation].jpg
        - [species]_leaf_[condition]_[number].jpg
        - etc.
        """
        # This is a template - adjust based on actual filenames
        return {
            'species_condition': re.compile(
```

## STEP 5: Build Master Manifest with Parent-Image Grouping

This is the **most critical** technical step. The master manifest ensures leakage-safe evaluation.

### 5.1 Create Manifest Builder Script

Create `src/tools/build_manifest.py`:

```python
#!/usr/bin/env python3
"""
Build the master manifest with parent-image grouping.
This is the foundation of leakage-safe evaluation.

Key principle: Every augmented image must map to exactly one original parent.
"""

import os
import pandas as pd
import hashlib
import re
from pathlib import Path
from PIL import Image
from tqdm import tqdm
from collections import defaultdict
import json

class ManifestBuilder:
    """Build leakage-safe manifest with parent-image tracking."""
    
    def __init__(self, raw_data_path, label_ontology_path):
        self.raw_data_path = raw_data_path
        self.records = []
        
        # Load label ontology
        with open(label_ontology_path) as f:
            self.ontology = json.load(f)
    
    def extract_parent_id(self, filename, folder_name):
        """
        Extract parent image ID from filename.
        
        The parent_id should be:
        - For original images: the full ID
        - For augmented images: the ID of the original it was derived from
        
        Adjust the regex based on actual filename patterns.
        """
        
        # Template patterns - adjust based on actual filenames
        # Example: leaf_001_healthy.jpg → parent_id = "leaf_001"
        # Example: leaf_001_healthy_aug_0.jpg → parent_id = "leaf_001"
        
        if 'Augmented' in folder_name:
            # Remove augmentation suffix
            parent = re.sub(r'_aug(mented)?.*$', '', filename, flags=re.IGNORECASE)
            parent = re.sub(r'_\d+\.(jpg|jpeg|png)$', f'.\\1', parent, flags=re.IGNORECASE)
        else:
            parent = filename
        
        return parent
    
    def build_original_index(self):
        """
        Create an index of all original images.
        Every augmented image must match an original.
        """
        originals = {}
        
        for root, dirs, files in os.walk(self.raw_data_path):
            if 'Original' in root and 'Augmented' not in root:
                for filename in files:
                    if filename.lower().endswith(('.jpg', '.jpeg', '.png')):
                        parent_id = self.extract_parent_id(filename, root)
                        originals[parent_id] = {
                            'filename': filename,
                            'folder': os.path.basename(root),
                            'path': os.path.join(root, filename)
                        }
        
        return originals
    
    def build_manifest(self, ontology_file='reports/label_ontology.json'):
        """Build complete manifest with parent-image tracking."""
        
        print("Building original image index...")
        original_index = self.build_original_index()
        print(f"Found {len(original_index)} original images")
        
        print("\nScanning all images and building manifest...")
        
        for root, dirs, files in os.walk(self.raw_data_path):
            folder_name = os.path.basename(root)
            is_original = 'Augmented' not in folder_name
            is_background_removed = 'Background_Removed' in folder_name
            
            for filename in tqdm(files, desc=folder_name):
                if not filename.lower().endswith(('.jpg', '.jpeg', '.png')):
                    continue
                
                filepath = os.path.join(root, filename)
                relative_path = os.path.relpath(filepath, self.raw_data_path)
                
                # Extract parent ID
                parent_id = self.extract_parent_id(filename, folder_name)
                
                # Verify parent exists (for augmented images)
                parent_found = parent_id in original_index or is_original
                
                record = {
                    'filename': filename,
                    'relative_path': relative_path,
                    'folder': folder_name,
                    'is_original': is_original,
                    'is_background_removed': is_background_removed,
                    'parent_leaf_id': parent_id,
                    'parent_found': parent_found,
                    'species': None,  # To be filled from ontology
                    'condition': None,  # To be filled from ontology
                    'file_size_bytes': os.path.getsize(filepath),
                }
                
                # Try to extract species and condition from filename
                # (adjust based on actual naming pattern)
                parts = filename.lower().split('_')
                
                # Match species
                for sp in self.ontology['species']:
                    if sp.lower() in filename.lower():
                        record['species'] = sp
                        break
                
                # Match condition
                for cond in self.ontology['conditions']:
                    if cond.lower() in filename.lower():
                        record['condition'] = cond
                        break
                
                self.records.append(record)
        
        return pd.DataFrame(self.records)
    
    def validate_manifest(self, df):
        """Perform integrity checks on the manifest."""
        
        errors = []
        warnings = []
        
        # Check 1: All parent IDs should map to an original
        unmapped_parents = df[~df['parent_found']]['parent_leaf_id'].unique()
        if len(unmapped_parents) > 0:
            errors.append(f"Found {len(unmapped_parents)} unmapped parent IDs: {unmapped_parents[:5].tolist()}")
        else:
            print("✓ All parent-image mappings are valid")
        
        # Check 2: Original images should be unique
        originals = df[df['is_original']]
        duplicate_originals = originals[originals.duplicated(subset=['parent_leaf_id'], keep=False)]
        if len(duplicate_originals) > 0:
            errors.append(f"Found {len(duplicate_originals)} duplicate original images")
        else:
            print("✓ All original images are unique")
        
        # Check 3: Check label coverage
        missing_species = df[df['species'].isna()]
        missing_condition = df[df['condition'].isna()]
        
        if len(missing_species) > 0:
            warnings.append(f"Could not extract species for {len(missing_species)} files")
        if len(missing_condition) > 0:
            warnings.append(f"Could not extract condition for {len(missing_condition)} files")
        
        print(f"\n✓ Species coverage: {(1 - len(missing_species)/len(df)):.1%}")
        print(f"✓ Condition coverage: {(1 - len(missing_condition)/len(df)):.1%}")
        
        # Check 4: Class distribution
        print(f"\nSpecies distribution:")
        print(df['species'].value_counts())
        print(f"\nCondition distribution:")
        print(df['condition'].value_counts())
        
        return errors, warnings
    
    def save_manifest(self, df, output_path='manifests/master_manifest.parquet'):
        """Save manifest in parquet format (compressed, preserves types)."""
        
        os.makedirs(os.path.dirname(output_path), exist_ok=True)
        df.to_parquet(output_path, index=False)
        print(f"\nMaster manifest saved to: {output_path}")
        print(f"Total records: {len(df)}")
        print(f"Columns: {df.columns.tolist()}")
        
        # Also save as CSV for inspection
        csv_path = output_path.replace('.parquet', '.csv')
        df.to_csv(csv_path, index=False)
        print(f"CSV version saved to: {csv_path}")
        
        return output_path

def main():
    print("\n" + "="*80)
    print("BUILDING MASTER MANIFEST WITH PARENT-IMAGE GROUPING")
    print("="*80 + "\n")
    
    builder = ManifestBuilder(
        raw_data_path='data/raw',
        label_ontology_path='reports/label_ontology.json'
    )
    
    # Build manifest
    df = builder.build_manifest()
    
    print(f"\nManifest built for {len(df)} files\n")
    
    # Validate
    errors, warnings = builder.validate_manifest(df)
    
    if errors:
        print("\n❌ ERRORS (must fix before proceeding):")
        for err in errors:
            print(f"  - {err}")
    
    if warnings:
        print("\n⚠ WARNINGS (review carefully):")
        for warn in warnings:
            print(f"  - {warn}")
    
    if not errors:
        # Save manifest
        output_path = builder.save_manifest(df)
        print("\n" + "="*80)
        print("MANIFEST BUILDING COMPLETE")
        print("="*80)
    else:
        print("\n⚠ Fix errors before proceeding!")

if __name__ == "__main__":
    main()
```

```python
#!/usr/bin/env python3
"""
Comprehensive integrity checks for the master manifest.
Run this before creating splits.
"""

import pandas as pd
import numpy as np
import json
from pathlib import Path

class ManifestChecker:
    """Verify manifest integrity and data quality."""
    
    def __init__(self, manifest_path):
        self.df = pd.read_parquet(manifest_path)
        self.errors = []
        self.warnings = []
        self.report = {}
    
    def check_parent_mapping(self):
        """Verify all parents map correctly."""
        print("Checking parent-image mappings...")
        
        # Check 1: All parent IDs should be mappable
        unmapped = self.df[~self.df['parent_found']]
        if len(unmapped) > 0:
            self.errors.append(f"{len(unmapped)} files have unmapped parents")
        else:
            print("  ✓ All parents mapped correctly")
        
        # Check 2: Each original should have consistent augmentations
        for parent_id in self.df['parent_leaf_id'].unique():
            group = self.df[self.df['parent_leaf_id'] == parent_id]
            species = group['species'].dropna().unique()
            condition = group['condition'].dropna().unique()
            
            if len(species) > 1:
                self.errors.append(f"Parent {parent_id} has inconsistent species: {species}")
            if len(condition) > 1:
                self.errors.append(f"Parent {parent_id} has inconsistent conditions: {condition}")
        
        if not self.errors:
            print("  ✓ All augmented images inherit correct labels from parents")
    
    def check_label_coverage(self):
        """Check label completeness."""
        print("Checking label coverage...")
        
        missing_species = self.df['species'].isna().sum()
        missing_condition = self.df['condition'].isna().sum()
        
        if missing_species > 0:
            self.warnings.append(f"{missing_species} files missing species labels ({missing_species/len(self.df):.1%})")
        if missing_condition > 0:
            self.warnings.append(f"{missing_condition} files missing condition labels ({missing_condition/len(self.df):.1%})")
        
        if missing_species == 0 and missing_condition == 0:
            print("  ✓ 100% label coverage")
    
    def check_class_balance(self):
        """Report class distribution."""
        print("Checking class distribution (at parent level)...")
        
        # Use only original images for true class distribution
        originals = self.df[self.df['is_original']].copy()
        
        species_dist = originals['species'].value_counts()
        condition_dist = originals['condition'].value_counts()
        
        self.report['species_distribution'] = species_dist.to_dict()
        self.report['condition_distribution'] = condition_dist.to_dict()
        
        # Check for extremely imbalanced classes
        if species_dist.min() / species_dist.max() < 0.1:
            self.warnings.append(f"Species classes highly imbalanced (ratio < 0.1)")
        
        if condition_dist.min() / condition_dist.max() < 0.1:
            self.warnings.append(f"Condition classes highly imbalanced (ratio < 0.1)")
        
        print(f"\n  Species distribution (among {len(originals)} originals):")
        for sp, count in species_dist.items():
            print(f"    {sp}: {count} ({count/len(originals)*100:.1f}%)")
        
        print(f"\n  Condition distribution:")
        for cond, count in condition_dist.items():
            print(f"    {cond}: {count} ({count/len(originals)*100:.1f}%)")
    
    def check_file_integrity(self):
        """Check that all files exist and have reasonable sizes."""
        print("Checking file integrity...")
        
        missing_files = []
        suspicious_sizes = []
        
        for idx, row in self.df.iterrows():
            filepath = Path('data/raw') / row['relative_path']
            
            if not filepath.exists():
                missing_files.append(row['filename'])
            
            # Check file size (very small or very large files)
            size_kb = row['file_size_bytes'] / 1024
            if size_kb < 10 or size_kb > 5000:  # Adjust based on typical sizes
                suspicious_sizes.append((row['filename'], size_kb))
        
        if missing_files:
            self.errors.append(f"{len(missing_files)} files missing from disk")
        else:
            print("  ✓ All manifest files exist on disk")
        
        if suspicious_sizes:
            if len(suspicious_sizes) > 0.01 * len(self.df):  # More than 1% suspicious
                self.warnings.append(f"Found {len(suspicious_sizes)} files with unusual sizes")
        else:
            print("  ✓ All file sizes look reasonable")
    
    def run_all_checks(self):
        """Run complete integrity check suite."""
        self.check_parent_mapping()
        self.check_label_coverage()
        self.check_class_balance()
        self.check_file_integrity()
    
    def print_report(self):
        """Print formatted check report."""
        print("\n" + "="*80)
        print("MANIFEST INTEGRITY CHECK REPORT")
        print("="*80 + "\n")
        
        print(f"Total files in manifest: {len(self.df)}")
        print(f"Original images: {self.df['is_original'].sum()}")
        print(f"Augmented images: {(~self.df['is_original']).sum()}")
        
        if self.errors:
            print(f"\n❌ CRITICAL ERRORS ({len(self.errors)}):")
            for err in self.errors:
                print(f"  - {err}")
        else:
            print(f"\n✓ No critical errors found")
        
        if self.warnings:
            print(f"\n⚠ WARNINGS ({len(self.warnings)}):")
            for warn in self.warnings:
                print(f"  - {warn}")
        else:
            print(f"\n✓ No warnings")
        
        print("\n" + "="*80)
        
        return len(self.errors) == 0

def main():
    checker = ManifestChecker('manifests/master_manifest.parquet')
    checker.run_all_checks()
    success = checker.print_report()
    
    if success:
        print("✓ MANIFEST INTEGRITY VERIFIED - Ready for split creation")
    else:
        print("❌ Fix errors before proceeding to next step")
    
    return success

if __name__ == "__main__":
    success = main()
    exit(0 if success else 1)
```

## STEP 7: Create Leakage-Safe Grouped Splits

### 7.1 Build Split Generator Script

Create `src/tools/make_grouped_splits.py`:

```python
#!/usr/bin/env python3
"""
Create leakage-safe, grouped, stratified train/val/test splits.

CRITICAL: This uses parent_leaf_id to ensure augmented siblings never cross boundaries.
"""

import pandas as pd
import numpy as np
from sklearn.model_selection import StratifiedKFold, train_test_split
import os
import json

class GroupedSplitGenerator:
    """Generate leakage-safe grouped splits."""
    
    def __init__(self, manifest_path, random_seed=3407):
        self.df = pd.read_parquet(manifest_path)
        self.seed = random_seed
        np.random.seed(random_seed)
        
        # Work with originals only
        self.originals = self.df[self.df['is_original']].copy()
        print(f"Working with {len(self.originals)} original images")
    
    def create_grouped_test_set(self, test_ratio=0.15):
        """
        Split at parent-image level, not file level.
        This ensures no data leakage.
        """
        print(f"\nCreating grouped test set ({test_ratio:.0%})...")
        
        unique_parents = self.originals['parent_leaf_id'].unique()
        print(f"Unique parent images: {len(unique_parents)}")
        
        # Get one representative record per parent
        parent_samples = self.originals.drop_duplicates(subset=['parent_leaf_id']).copy()
        
        # Stratified split at parent level
        test_parents, dev_parents = train_test_split(
            parent_samples,
            test_size=test_ratio,
            random_state=self.seed,
            stratify=parent_samples['condition']  # Or 'species' if too imbalanced
        )
        
        test_parent_ids = set(test_parents['parent_leaf_id'].values)
        dev_parent_ids = set(dev_parents['parent_leaf_id'].values)
        
        # Create test set (all files from test parents)
        test_set = self.df[self.df['parent_leaf_id'].isin(test_parent_ids)].copy()
        dev_pool = self.df[self.df['parent_leaf_id'].isin(dev_parent_ids)].copy()
        
        print(f"  Test parent groups: {len(test_parent_ids)}")
        print(f"  Dev parent groups: {len(dev_parent_ids)}")
        print(f"  Test files: {len(test_set)}")
        print(f"  Dev files: {len(dev_pool)}")
        
        return test_set, dev_pool, test_parent_ids, dev_parent_ids
    
    def create_grouped_cv_folds(self, dev_pool, n_folds=5):
        """
        Create grouped k-fold cross-validation at parent level.
        """
        print(f"\nCreating grouped {n_folds}-fold cross-validation...")
        
        # Get unique parents in dev pool
        dev_parents = dev_pool.drop_duplicates(subset=['parent_leaf_id']).copy()
        
        # Stratified k-fold at parent level
        skf = StratifiedKFold(n_splits=n_folds, shuffle=True, random_state=self.seed)
        
        folds = []
        for fold_idx, (train_indices, val_indices) in enumerate(skf.split(dev_parents, dev_parents['condition'])):
            train_parents = dev_parents.iloc[train_indices]['parent_leaf_id'].values
            val_parents = dev_parents.iloc[val_indices]['parent_leaf_id'].values
            
            # Get all files (including augmentations) for each group
            fold_train = dev_pool[dev_pool['parent_leaf_id'].isin(train_parents)].copy()
            fold_val = dev_pool[dev_pool['parent_leaf_id'].isin(val_parents)].copy()
            
            folds.append({
                'fold': fold_idx,
                'train': fold_train,
                'val': fold_val,
                'train_parent_ids': set(train_parents),
                'val_parent_ids': set(val_parents)
            })
            
            print(f"  Fold {fold_idx}: train={len(fold_train)} files ({len(train_parents)} parents), "
                  f"val={len(fold_val)} files ({len(val_parents)} parents)")
        
        return folds
    
    def verify_no_leakage(self, test_set, dev_pool, folds):
        """Verify there is no parent-image leakage across splits."""
        print("\n" + "="*80)
        print("VERIFYING NO DATA LEAKAGE")
        print("="*80)
        
        test_parents = set(test_set['parent_leaf_id'].values)
        
        # Check test vs dev
        overlap = test_parents & set(dev_pool['parent_leaf_id'].values)
        if overlap:
            print(f"❌ ERROR: {len(overlap)} parents in both test and dev!")
            return False
        else:
            print("✓ Test and dev sets have no parent overlap")
        
        # Check each fold
        for fold in folds:
            train_parents = fold['train_parent_ids']
            val_parents = fold['val_parent_ids']
            
            overlap = train_parents & val_parents
            if overlap:
                print(f"❌ ERROR in fold {fold['fold']}: {len(overlap)} parents in both train and val!")
                return False
            
            # Also check against test
            overlap_test = train_parents & test_parents
            if overlap_test:
                print(f"❌ ERROR in fold {fold['fold']}: {len(overlap_test)} parents in both train and test!")
                return False
            
            overlap_test_val = val_parents & test_parents
            if overlap_test_val:
                print(f"❌ ERROR in fold {fold['fold']}: {len(overlap_test_val)} parents in both val and test!")
                return False
        
        print(f"✓ All {len(folds)} folds verified - zero parent leakage")
        return True
    
    def save_splits(self, test_set, folds, output_dir='manifests/splits_v1'):
        """Save all splits to CSV files."""
        os.makedirs(output_dir, exist_ok=True)
        
        print(f"\nSaving splits to {output_dir}...")
        
        # Save test set
        test_output = os.path.join(output_dir, 'test_groups.csv')
        test_set.to_csv(test_output, index=False)
        print(f"  Saved: {test_output} ({len(test_set)} files)")
        
        # Save each fold
        for fold in folds:
            fold_idx = fold['fold']
            
            train_output = os.path.join(output_dir, f'dev_fold_{fold_idx}_train.csv')
            val_output = os.path.join(output_dir, f'dev_fold_{fold_idx}_val.csv')
            
            fold['train'].to_csv(train_output, index=False)
            fold['val'].to_csv(val_output, index=False)
            
            print(f"  Saved fold {fold_idx}: train={len(fold['train'])}, val={len(fold['val'])}")
        
        # Create split report
        report = {
            'random_seed': self.seed,
            'test_ratio': len(test_set) / len(self.df),
            'n_folds': len(folds),
            'total_files': len(self.df),
            'test_files': len(test_set),
            'dev_files': len(pd.concat([f['train'] for f in folds] + [f['val'] for f in folds]).drop_duplicates(subset=['filename'])),
            'test_parent_groups': len(set(test_set['parent_leaf_id'].values)),
            'verification': 'zero_parent_leakage'
        }
        
        report_output = os.path.join(output_dir, 'split_report.json')
        with open(report_output, 'w') as f:
            json.dump(report, f, indent=2)
        
        print(f"  Saved: {report_output}")
        
        return output_dir

def main():
    print("\n" + "="*80)
    print("CREATING LEAKAGE-SAFE GROUPED SPLITS")
    print("="*80)
    
    generator = GroupedSplitGenerator(
        manifest_path='manifests/master_manifest.parquet',
        random_seed=3407
    )
    
    # Create test set and dev pool
    test_set, dev_pool, test_parents, dev_parents = generator.create_grouped_test_set(test_ratio=0.15)
    
    # Create folds
    folds = generator.create_grouped_cv_folds(dev_pool, n_folds=5)
    
    # Verify no leakage
    if not generator.verify_no_leakage(test_set, dev_pool, folds):
        print("\n❌ LEAKAGE DETECTED - ABORTING")
        return False
    
    # Save splits
    generator.save_splits(test_set, folds)
    
    print("\n" + "="*80)
    print("GROUPED SPLITS CREATED SUCCESSFULLY")
    print("="*80 + "\n")
    
    return True

if __name__ == "__main__":
    success = main()
    exit(0 if success else 1)
```
