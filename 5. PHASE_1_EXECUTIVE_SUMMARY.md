### Objective 3: Build Master Manifest with Parent-Image Grouping
**Timeline:** Days 8
**Goal:** Create a definitive inventory with parent-image relationships

**The Critical Principle:**
Every file must map to exactly one **parent original image**. Augmented images inherit the parent's labels. This prevents leakage.

**Deliverables:**
```
✓ manifests/master_manifest.parquet  - Binary manifest (compressed)
✓ manifests/master_manifest.csv      - Human-readable version
```

**Manifest Structure (Key Columns):**
```
filename              - Original filename
relative_path        - Path within data/raw
folder               - Which folder (Original/Augmented/Background_Removed)
is_original          - Boolean: is this an original image?
is_background_removed - Boolean: is background removed?
parent_leaf_id       - CRITICAL: ID of the original parent
parent_found         - Boolean: does the parent exist?
species              - Species label (from parent)
condition            - Health condition (from parent)
file_size_bytes      - File size
```

**Acceptance Criteria:**
- All files indexed (should be ~22,000)
- All parent mappings valid (no unmapped parents)
- All augmented images map to existing originals
- Label consistency verified
- Zero duplicates in parent_leaf_id field

### Objective 5: Create Leakage-Safe Grouped Splits
**Timeline:** Days 9-10
**Goal:** Split the data in a way that prevents augmentation leakage

**The Split Strategy:**

1. **Test Set:** 15% of original parents (~297 parents)
   - Held-out, untouched until final evaluation
   - No augmentations in test set (use originals only)

2. **Development Pool:** 85% of original parents (~1,684 parents)
   - Used for model selection and hyperparameter tuning
   - Split into 5-fold cross-validation

3. **Augmentation Policy:**
   - Generate augmentations **AFTER** splitting
   - Augmentations only in training loop
   - Validation and test: resize/pad/normalize only (no augmentation)

**Deliverables:**
```
✓ manifests/splits_v1/test_groups.csv           - Test set manifest
✓ manifests/splits_v1/dev_fold_0_train.csv     - Fold 0 training set
✓ manifests/splits_v1/dev_fold_0_val.csv       - Fold 0 validation set
✓ manifests/splits_v1/dev_fold_1_train.csv     - Fold 1 training set
✓ manifests/splits_v1/dev_fold_1_val.csv       - [etc. x 5 folds]
✓ manifests/splits_v1/split_report.json        - Split metadata
✓ reports/split_report.md                      - Split documentation
```

**Acceptance Criteria (CRITICAL):**
- ✅ Test set: ~297 parents, ~297+ files (no augmentations)
- ✅ Dev pool: ~1,684 parents in 5 stratified folds
- ✅ **ZERO parent-image leakage** (verified by script)
  - No parent appears in multiple splits
  - No parent appears in both train and val within same fold
  - No parent appears in both dev pool and test set
- ✅ Stratification maintained (class balance preserved)
- ✅ All splits are **immutable** after this point

**Verification Command:**
```bash
# This MUST return zero leakage
python src/tools/check_no_leakage.py
```

## KEY TECHNICAL DECISIONS (LOCKED IN PHASE 1)

| Decision | Value | Rationale |
|----------|-------|-----------|
| **Grouping Unit** | parent_leaf_id (original image) | Prevents augmentation leakage |
| **Source Images** | Background-removed originals (1440×1080) | Reduces confounding, preserves leaf morphology |
| **Test Ratio** | 15% of parents (~297) | Stable, provides meaningful test set |
| **CV Strategy** | 5-fold stratified on parents | Standard, enables robust model selection |
| **Stratification** | By condition (or species if condition too imbalanced) | Ensures all classes in all folds |
| **Random Seed** | 3407 | Reproducibility (never change after splits created) |
| **Augmentation** | Train-only, restrained (rotation ±20°, flip, jitter) | Prevents overfitting without erasing disease cues |
| **Label Ontology** | 7 species × ~8-9 conditions | Granular enough for health detection |

**⚠️ These decisions are NON-NEGOTIABLE after Phase 1 completion**

## COMMON MISTAKES TO AVOID

| ❌ Mistake | ✅ Correct Approach |
|-----------|------------------|
| Using file-level splits | Use parent-image grouping |
| Including augmentations in test set | Test set contains originals only |
| Creating augmentations before splitting | Split first, augment only training data |
| Regenerating splits after modeling | Freeze splits, never change after locked |
| Using inconsistent label definitions | Freeze ontology before manifest building |
| Incomplete documentation | Document decisions and dates immediately |
| Testing hypothesis before splits locked | Lock splits first, then design experiments |
| Pushing to Phase 2 with unverified data | Run ALL integrity checks before proceeding |
