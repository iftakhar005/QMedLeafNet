## 🎯 Core Concept: Why Phase 1 Matters

**The Problem:**
Your dataset has:
- ✓ 1,981 **original** images (unique leaf samples)
- ✓ 20,019 **augmented** images (derived from originals)

If augmented images leak across train/test boundaries → **ALL RESULTS INVALID**

**The Solution (Phase 1):**
1. Group by parent image (not filename)
2. Create leakage-safe splits at parent level
3. Augment AFTER splitting (training only)
4. Verify ZERO parent leakage
5. Freeze everything before modeling

## 🔴 Common Mistakes (AVOID THESE)

| ❌ Mistake | ✅ Correct |
|-----------|----------|
| File-level splits | Parent-level splits |
| Augment before split | Split then augment |
| Change splits mid-project | Freeze splits once |
| Include augmented images in test | Test = originals only |
| Skip leakage verification | Run 3x verification |
| Recompute splits later | Immutable after locking |

## 📞 If You Get Stuck

### Problem: Parent mapping fails
→ Check filename patterns in your dataset  
→ Adjust regex in `extract_metadata.py`  
→ Verify consistency: `find data/raw -name "*.jpg" | head -10`

### Problem: Leakage detected
→ **STOP immediately**  
→ Re-examine parent_leaf_id extraction  
→ Check manifest: `python3 -c "import pandas as pd; df = pd.read_parquet('manifests/master_manifest.parquet'); print(df[df['parent_found']==False])"`

### Problem: Labels not extracted
→ Manually examine filenames: `ls data/raw/Original_Images/ | head -20`  
→ Check if species/condition are in filenames  
→ Adjust parsing logic
